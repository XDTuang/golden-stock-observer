## 二、本地验证 + 微调参数（`--dry-run` 实操）

脚本只依赖一个 Python 库 `requests`，你自己的 Mac 就够了，**不需要**那个很长的 managed 路径。

### 准备（只需做一次）
1. 打开「终端」：按 `Cmd+空格` 输入 `终端` 回车。
2. 进到项目目录（复制粘贴回车）：
   ```bash
   cd /Users/samt/golden_stock_observer/repurchase-yield
   ```
3. 装依赖（若已装过会秒过）：
   ```bash
   pip3 install requests
   ```

### 步骤 1：先跑"只看不写"的干跑
```bash
python3 update_repurchase.py --dry-run
```
你会看到类似输出：
```
[info] 数据窗口 2026-07-19 ~ 2026-07-20 (每日 20:00 触发)
[info] 新浪命中含'回购目标价'公告 N 条
[info] 可计算隐含收益率 M 条
[
  {"stock": "...", "code": "...", "target": ..., "close": ..., ...},
  ...
]
```
- **M > 0 且能看到真实股票名 + 目标价 + 收益率** → 链路通了，参数不用动。
- **N=0 / M=0** → 窗口内本就没新回购公告（非交易日/平静期常见，正常）；若连续多日 0 且你确定有回购，检查网络或被限流（见下方"微调点"）。
- **想看全链路（含目标价从正文抠取）**，扩窗口验证：
  ```bash
  python3 update_repurchase.py --from 2026-06-01 --to 2026-07-21 --dry-run
  ```
  会抓到例如「莲花控股 600186.SH 目标价16.04 / 收盘8.14 / 隐含+97.1%」这类真实记录。

### 步骤 2：回填历史区间测试（验证抓取+解析全链路）
```bash
python3 update_repurchase.py --from 2026-07-01 --to 2026-07-20 --dry-run
```
这会用 7/1–7/20 的窗口跑一遍，方便你确认能抓到那 13 条样本附近的数据。

### 微调点（脚本里改这几处，改完重新 `--dry-run`）
| 想调什么 | 在 `update_repurchase.py` 改哪行 | 说明 |
|---|---|---|
| **数据源切换** | 运行时 `--source sina`（默认）/ `--source cninfo` | 默认**新浪**（稳定可解析正文）；巨潮仅作备选，家用宽带常被风控 500 |
| 抓取接口(新浪) | `SINA_BULLETIN = "https://money.finance.sina.com.cn/..."` | 新浪个股公告页，按股票池逐股扫 |
| 过滤"非目标价类"回购 | `REPO_EXCLUDE = (...)` | 含「注销/减资/通知债权人/限制性股票/减持已回购/法律意见书…」的回购公告不计入（无隐含收益率意义） |
| 目标价解析正则 | `PRICE_RES = [ ... ]` | **已从新浪公告正文(HTML)抠"回购价格上限 X 元/股"**，多模式兜底；无需再下钻 PDF |
| 股票池(扫哪些股) | `candidate_pool.json`（默认）或 `--pool /path/pool.json` | 当前是 64 只策略池；换成你的 Top800 名单即可扩大覆盖 |
| 请求头伪装 | `UA`、`http_get` 里的 `headers` | 浏览器 UA + Referer，降低被识破概率 |
| 随机延时 | `MIN_DELAY, MAX_DELAY = 0.6, 2.4` | 越大越"像人"，但越慢（扫 64 只约 1–2 分钟） |
| 重试次数 | `MAX_RETRY = 3` | 失败指数退避重试 |
| 代理池 | 运行前 `export REPO_PROXY=http://host:port` 或加 `--proxy http://host:port` | 数据中心 IP 被拦时的兜底 |

> ✅ 真实"回购目标价"现已能**下钻新浪公告正文自动提取**（如「回购价格上限 16.04元/股」），不再只靠标题估算。
> 极少数公告若用非常规表述导致未命中，会被跳过（不影响其他记录）。

## 三、真正的"每日自动抓取 + 反拦截 + 定时重生成"怎么落地

### 先说原理：为什么光有网页不行
浏览器打开的网页**不能**自己定时去抓数据、改文件——它得有个"后台机器人"按点干活。
这个机器人要么是一台常开服务器，要么是 GitHub 的免费定时任务（Actions）。
我们选了 **GitHub Actions**：免费、不用维护服务器、和仓库一体。

### 方案 A：GitHub Actions（已为你配好 ✅）
文件：`.github/workflows/repurchase-daily.yml`
- **触发时间**：每天 **北京 20:00**（cron `0 12 * * *`，UTC 12:00 = 北京 20:00）。
- **手动也能跑**：仓库页 → 顶部 **Actions** → 找「回购隐含收益 · 每日更新」→ 右侧 **Run workflow**。
- **它每天自动做**：
  1. 拉取仓库 → 装 Python + requests
  2. 跑 `update_repurchase.py`（抓前一日20:00→当日20:00的回购公告）
  3. 算隐含收益率、按 `(代码,公告日期)` 去重、累加进 `records.json`
  4. 重写 `index.html` 的 DATA 块
  5. 有变化就自动 `commit + push` → GitHub Pages 自动上线
- **反拦截在哪**：全在 `update_repurchase.py` 里（UA/Referer 头、随机延时、指数退避重试、可选代理）。机器人只是按时调用它。

#### 你怎么看它跑没跑 / 出没出错
1. 打开 `https://github.com/XDTuang/golden-stock-observer` → 点 **Actions** 标签。
2. 列表里每条是一次运行；点进去看日志（绿勾=成功，红叉=失败）。
  3. 失败先看日志最后几十行：
   - `巨潮命中 0 条` 或 `HTTP 500` → 说明脚本跑在 `--source cninfo` 上且被拦；默认已是新浪源，确认没加 `--source cninfo` 即可。
   - `新浪命中 0 条` 但没报错 → 窗口内本就没新的"集中竞价设目标价"类回购（正常，平静期常见）。
   - `无收盘价` / 网络超时 → 新浪或腾讯行情接口临时抖动，重试或调大 `MAX_DELAY`/`MAX_RETRY`。

### 方案 B：腾讯云（备选，节奏独立于主站）
仓库里已有 `腾讯云托管自动更新方案.md`，给出 COS+CDN / SCF 定时触发器 / 轻量服务器三条路线。
SCF 方案最贴合"每日20:00、且节奏独立于主站其他数据"——云函数定时触发器直接调这个脚本写 COS。
**优先把 GitHub Actions 跑顺**，腾讯云等你想换/加备份时再上。

### 你的日常运维清单（很轻）
- ✅ **平时不用管**：机器人每天 20:00 自己跑、自己上线。
- 🔧 **发现某天没新数据**：去 Actions 看日志，多半是窗口内确实无新回购，或被限流 → 按"微调点"改脚本 → `git push` 即可生效（下次自动用新脚本）。
- 🔧 **想立刻补跑一天**：本地 `--from/--to` 干跑确认无误后，也可以手动在 Actions 点 Run workflow（脚本默认抓昨日→今日）。
- 🔧 **数据明显不对**（如目标价错）：基本是"标题估价格"的局限 → 需补"公告正文解析"（见 TODO）。
